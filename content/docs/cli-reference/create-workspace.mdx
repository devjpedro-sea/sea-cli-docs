# sea-cli create-workspace

Cria um novo workspace Liferay DXP do zero com todas as configurações de qualidade de código.

## Sintaxe

```bash
sea-cli create-workspace <nome>
```

## Parâmetros

### nome (obrigatório)

Nome do workspace a ser criado.

**Tipo:** String  
**Regras:**
- Apenas letras, números, hífens e underscores
- Não pode começar com número
- Recomendado: kebab-case

**Exemplos válidos:**
```bash
sea-cli create-workspace meu-projeto
sea-cli create-workspace portal_cliente
sea-cli create-workspace app2024
```

**Exemplos inválidos:**
```bash
sea-cli create-workspace 123-projeto  # Começa com número
sea-cli create-workspace meu projeto  # Tem espaço
sea-cli create-workspace meu@projeto  # Caractere inválido
```

## Descrição

O comando `create-workspace` automatiza a criação de um workspace Liferay completo:

1. Clona template base do repositório
2. Instala todas as dependências NPM
3. Configura ESLint + Prettier
4. Configura Husky + lint-staged
5. Configura Jest + Testing Library
6. Adiciona scripts de automação
7. Inicializa repositório Git

## O Que É Criado

### Estrutura de Diretórios

```
meu-projeto/
├── .husky/
│   └── pre-commit
├── modules/
├── client-extensions/
├── configs/
├── .eslintrc.js
├── .prettierrc.js
├── jest.config.js
├── babel.config.js
├── package.json
├── .gitignore
└── README.md
```

### Arquivos de Configuração

- `.eslintrc.js` - Regras ESLint
- `.prettierrc.js` - Formatação Prettier
- `jest.config.js` - Configuração de testes
- `babel.config.js` - Transpilação
- `.husky/pre-commit` - Git hook

```json
    "lint:format": "prettier --write .",
    "lint:fix": "eslint . --fix",
    "test": "jest",
    "test:coverage": "jest --coverage",
    "test:watch": "jest --watch"
  }
}
```

### Criar e Entrar

```bash
sea-cli create-workspace meu-app && cd meu-app
```

### Criar Múltiplos

```bash
# Criar vários workspaces
for proj in app1 app2 app3; do
  sea-cli create-workspace $proj

```

## Fluxo Completo

```bash
# 1. Criar workspace
sea-cli create-workspace ecommerce-portal

# 2. Entrar no workspace
cd ecommerce-portal

# 3. Verificar estrutura
ls -la

# 4. Testar configurações
npm run lint

# 5. Executar testes
npm test

# 6. Criar primeiro módulo
cd modules
sea-cli create-module

# 7. Fazer primeiro commit
git add .
git commit -m "feat: workspace inicial"
```

## Troubleshooting

### Erro: Diretório já existe

**Problema:**
```
Error: Directory meu-projeto already exists
```

**Solução:**
```bash
# Remover ou renomear
rm -rf meu-projeto
# OU
sea-cli create-workspace meu-projeto-v2
```

### Erro: npm install falhou

**Problema:**
```
Error: Failed to install dependencies
```

**Solução:**
```bash
cd meu-projeto
rm -rf node_modules package-lock.json
npm install
```

### Erro: Falha ao clonar

**Problema:**
```
Error: Failed to clone repository
```

**Solução:**
- Verificar conexão internet
- Verificar acesso ao GitLab
- Verificar token configurado

```bash
npm config get @sea-quality-code:registry
```

## Diferenças vs Setup

| Aspecto | create-workspace | setup |
|---------|------------------|-------|
| **Workspace** | Cria novo | Usa existente |
| **Estrutura** | Template completo | Adiciona configs |
| **Repositório** | Clona do Git | Usa local |
| **Uso** | Projetos novos | Projetos existentes |

## CI/CD

### GitHub Actions

```yaml
name: Create Workspace
on:
  workflow_dispatch:
    inputs:
      workspace_name:
        description: 'Nome do workspace'
        required: true

jobs:
  create:
    runs-on: ubuntu-latest
    steps:
      - name: Install SEA CLI
        run: npm i -g @sea-quality-code/sea-cli
        
      - name: Create workspace
        run: sea-cli create-workspace ${{ github.event.inputs.workspace_name }}
```

### Script Bash

```bash
#!/bin/bash
# create-workspace.sh

PROJECT_NAME=$1

if [ -z "$PROJECT_NAME" ]; then
  echo "Usage: ./create-workspace.sh <project-name>"
  exit 1
fi

sea-cli create-workspace $PROJECT_NAME
cd $PROJECT_NAME
npm run lint
npm test

echo "Workspace $PROJECT_NAME criado e testado!"
```
